# 📝 Technical & Product Documentation | `docs/`

---

## 🎯 Purpose
The documentation hub is the Single Source of Truth (SSOT) representing the entire technical design, business logic, design tokens, and third-party integrations of the system.

---

## 📂 Subdirectories
* **[database/](file://./database/)** — Security rules, entity schemas, and ORM setups.
* **[design/](file://./design/)** — Design systems, styling variables, and typography.
* **[features/](file://./features/)** — In-depth technical specifications of product modules.
* **[integrations/](file://./integrations/)** — Documentation on all webhooks, API tokens, and SDK integrations.
* **[project/](file://./project/)** — High-level architecture, user personas, and core specs.
* **[tech-stack/](file://./tech-stack/)** — Reference guides for packages, libraries, and runtimes.
