# 📘 Product & Technical Documentation Hub

This directory serves as the **Single Source of Truth (SSOT)** for all product specifications, architecture, Cloud Firestore schemas, design styling tokens, and API integrations of the FlutterFlow & Firebase project.

---

## 📂 Documentation Directory Map

```text
docs/
├── database/ ............. Firestore schemas, collection maps, and security rules
├── design/ ............... UI/UX visual design system, parameters, and tokens
├── features/ ............. Functional specifications and client flow diagrams
├── flutterflow/ .......... Custom Action and Custom Function mapping guidelines
├── integrations/ ......... Third-party REST APIs and Stripe configurations
├── project/ .............. Architectural context and overall product layouts
└── tech-stack/ ........... Dart language specs and library dependency whitelists
```

---

## 🛠️ Directory Breakdown

### 1. 🗄️ [database/](file://./database/)
* Firestore collections structure and schema mappings.
* Rules files outlining read/write security policies.

### 2. 🎨 [design/](file://./design/)
* Theme assets, brand colors, typography, and premium widgets guidelines.

### 3. ⚙️ [features/](file://./features/)
* Functional specs detailing step-by-step user journeys and layout specifications.

### 4. 📱 [flutterflow/](file://./flutterflow/)
* Best practices on building scalable widgets, parameter configurations, and actions logic.

### 5. 🔌 [integrations/](file://./integrations/)
* Integrations with Stripe, push notifications, and external client endpoints.

### 6. 🗺️ [project/](file://./project/)
* Component hierarchy maps and development flowcharts.

### 7. 🛠️ [tech-stack/](file://./tech-stack/)
* Dependency checklists, Dart versions, and custom packages whitelist.
