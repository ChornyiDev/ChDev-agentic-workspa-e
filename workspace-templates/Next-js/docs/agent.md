# 📝 Technical & Product Documentation | `docs/`

---

## 🎯 Purpose
The documentation hub is the Single Source of Truth (SSOT) representing the entire technical design, business logic, design tokens, and third-party integrations of the system.

---

## 📂 Subdirectories
* **[database/](file://./database/)** — Neon Postgres security rules, schemas (`001_schema_simple.sql`, `001_schema_full.sql`), and connection specifications.
* **[design/](file://./design/)** — Design systems, responsive styling guidelines, and typography.
* **[features/](file://./features/)** — Detailed technical and functional feature specifications.
* **[integrations/](file://./integrations/)** — Third-party webhooks, API credentials, and SDK configurations.
* **[project/](file://./project/)** — High-level architecture, roadmap alignments, and core overviews.
* **[tech-stack/](file://./tech-stack/)** — Package checklists, React 19 rules, and library versions.
